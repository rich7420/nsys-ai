# nsys-ai — Claude Code Skill

A Claude Code plugin that turns NVIDIA Nsight Systems GPU profiles into actionable
performance fixes. Point it at a `.sqlite` or `.nsys-rep` profile and get a root cause,
a specific code fix, and a quantified expected gain — in 3 turns or fewer.

## Install

```bash
# 1. Install the nsys-ai CLI (required)
pip install "nsys-ai[agent]"

# 2. Add the plugin to Claude Code
#    Option A — marketplace (once published)
claude plugin install GindaChen/nsys-ai

#    Option B — local dev (session-scoped)
git clone https://github.com/GindaChen/nsys-ai
claude --plugin-dir ./nsys-ai
```

Verify: `nsys-ai --help` should exit 0.

## Usage

Type `/nsys-ai` in Claude Code. The skill auto-detects profiles in your working
directory and shows a mode menu. You can also pass a profile or question directly.

```
/nsys-ai                                        # scan CWD, show menu
/nsys-ai profile.sqlite                         # auto-triage a single profile
/nsys-ai profile.sqlite why is my training slow # skip menu, direct question
/nsys-ai before.sqlite after.sqlite             # compare two profiles (Mode 8)
/nsys-ai profile.sqlite spikes                  # jump to variance analysis (Mode 9)
```

## Mode Menu

When the skill starts it renders this menu once per session:

```
What would you like to analyze?

  1. Auto triage      — not sure where to start                  [default]
  2. Comms            — NCCL / overlap / multi-GPU
  3. Compute          — top kernels / tensor core / MFU
  4. Memory           — H2D / D2H / bandwidth
  5. NVTX / code map  — which layer / step consumes time
  6. Idle / sync      — GPU gaps, CPU stalls, launch overhead
  7. CUTracer         — SASS-level (requires re-run)
  8. Diff             — compare two profiles
  9. Variance         — some iterations much slower than others

Reply with a number, or ask a question directly — keywords auto-route.
```

## Keyword Auto-routing

You never have to pick a number. Keywords in your message auto-select the right mode
(first match wins; common examples shown — see `SKILL.md §Keyword Routing` for the full list):

| Keywords (examples) | Mode |
|---------------------|------|
| `cutracer`, `sass`, `instruction-level`, `warp divergence` | 7 — CUTracer |
| `diff`, `compare`, `regression`, `before/after`, or 2 paths | 8 — Diff |
| `variance`, `spike`, `jitter`, `choppy`, `straggler` | 9 — Variance |
| `nccl`, `allreduce`, `allgather`, `reducescatter`, `overlap` | 2 — Comms |
| `memory`, `h2d`, `d2h`, `bandwidth`, `transfer`, `pin_memory` | 4 — Memory |
| `nvtx`, `layer`, `per-layer`, `iteration timing`, `code mapping` | 5 — NVTX |
| `idle`, `sync`, `launch overhead`, `dataloader`, `gap` | 6 — Idle |
| `gemm`, `flash`, `attention`, `kernel`, `tensor core`, `mfu`, `tflops`, `utilization` | 3 — Compute |
| `why`, `slow`, `bottleneck`, `?` | 1 — Auto triage |

## What you get

Every mode delivers a **3-part structured answer**:

1. **Root cause** — mechanism + magnitude (e.g. "`flash_bwd` accounts for 82% share, +58 ms after the change")
2. **Specific fix** — matched to the diagnosis (e.g. `DDP(model, bucket_cap_mb=256)`)
3. **Expected gain** — quantified (e.g. "should recover ~58 ms per step")

Plus an **interactive timeline URL** (`http://127.0.0.1:PORT`) with the bottleneck
highlighted, generated by `nsys-ai timeline-web`.

## Requirements

- Python 3.10+
- `nsys-ai >= 0.2.1` (`pip install "nsys-ai[agent]"`)
- NVIDIA Nsight Systems for capturing profiles (not needed for analysis)
- Mode 7 (CUTracer) additionally requires `cutracer.so` + `nvdisasm` (optional)
